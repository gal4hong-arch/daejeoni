행정 AI

무엇이든 물어보세요

생성형 AI는 실수를 할 수 있습니다. 법령·행정 판단은 반드시 공식 문서와 담당 부서 확인을 병행하세요.

채팅에서 마지막으로 잡힌 토픽을 기준으로 통합 보고서 초안을 만듭니다. (설명자료·의회 대응·질의 시뮬 요소를 한 문서 흐름으로 포함)

통합 보고서·양식·체인 설명
통합 보고서
개요·주요 내용·상급 검토 포인트에 더해, 설명자료만의 축(목적·근거·추진·효과), 의회 대응 축(예상 질의·답변 초안·향후 계획), 의회·상급 질의 시뮬 블록을 한 번씩만 다루도록 통합합니다. 중복 문단은 피합니다.
시나리오 힌트
비우면 반영하지 않습니다. 적으면 보고서 작성(및 API 시뮬 체인)에 각도·쟁점으로 넘깁니다.
서식 양식 파일
HWP·HWPX·DOC·DOCX를 드래그 앤 드롭(또는 클릭 선택)하면 텍스트를 추출해 목차·항목에 맞춰 본문을 채웁니다. 보고서 생성 시 항상 작성 보강 → 상급자 검토 → 법령 점검 순 3단 에이전트가 실행되며, 양식이 있으면 각 단계에 반영됩니다.
Word 다운로드
양식을 사용해 생성한 경우(서버에서 양식 반영이 적용된 경우) Word로 다운로드로 양식에 맞춘 작성(보강) 본문을 .docx로 받을 수 있습니다.
양식을 여기에 놓거나 클릭하여 선택
HWP · HWPX · DOC · DOCX
양식 선택됨 · 보고서 생성 시 사용
다른 파일을 놓으면 교체됩니다.
「보고서」를 누르면 이 영역이 비운 뒤 「생성 중입니다…」가 표시되고, 완료되면 초안이 나옵니다. 형식은 채팅 답변과 같습니다.

검색에 쓸 소스를 목록에서 선택하면 채팅 시 해당 문서의 청크만 하이브리드 검색에 사용합니다. 이름 줄을 클릭해 켜고 끕니다.

등록된 소스 목록

내가 등록한 소스는 목록에서 오른쪽 클릭하면 표시 제목을 바꿀 수 있습니다. 동일 URL·동일 파일 내용을 다시 올리면 기존 청크를 교체합니다.

자주 찾는 법령

채팅에서 법령 연동으로 실제 답변에 쓰인 법령이 사용자별로 누적됩니다(조회 횟수 순). 목록의 ID는 링크·검색 조문이면 일련번호(MST)인 경우가 많고, 법령 소스 등록에는 본문 API의 법령ID를 넣습니다. 같은 법령이라도 두 숫자가 다를 수 있으며, 등록 후 제목·메타에 둘 다 표시됩니다. (OpenAI 키 필요)

소스 파일 (PDF·문서·이미지)

아래 영역에 드래그 앤 드롭하거나 파일을 고른 뒤 파일 소스 등록을 누르세요. PDF는 기본으로 텍스트 레이어만 추출합니다(스캔 PDF는 글자가 없을 수 있음). 필요할 때만 텍스트+OCR을 고르세요.

텍스트 레이어만 (기본)
PDF에 대해 OCR 생략 · 대용량은 쪽 단위로 나눠 처리
텍스트 + OCR
글자 레이어 + 필요 시 이미지 OCR · 40쪽 이하 전 쪽 OCR 시도
여기에 파일을 놓거나 클릭하여 선택
PDF, TXT, MD, 이미지(PNG·JPEG·WEBP·GIF 등), PPT, PPTX, XLS, XLSX, DOC, DOCX, HWP, HWPX
등록 준비
아래 「파일 소스 등록」을 누르면 서버에 등록됩니다.
웹 페이지 (URL)

페이지의 제목(title)이 소스 이름으로 자동 설정됩니다. 같은 URL을 다시 가져오면 해당 소스의 청크를 교체합니다.

본문 직접 입력

비어 있지 않은 첫 줄을 제목 후보로 씁니다. 없으면 「직접 입력 소스」로 표시됩니다.

안내
  1. 자주 찾는 법령 → 행에서 법령 소스 등록.
  2. 소스 파일 → 드롭 또는 파일 선택 후 파일 소스 등록(위 확장자).
  3. URL → 주소 입력 후 URL 소스 등록.
  4. 직접 입력 → 본문 입력 후 소스 등록.
  5. 채팅에서는 목록에서 선택한 소스만 검색 범위로 씁니다(선택 상태는 브라우저에 저장).

위에 없는 확장자는 등록할 수 없으며, 지원 형식이라도 서버에 변환기가 없으면 안내 메시지가 표시될 수 있습니다. 그 경우 PDF나 TXT로 저장한 뒤 등록하세요.

내가 보낸 질문은 아래 목록에 모입니다. 항목을 누르면 채팅으로 이동합니다. 대분류·중분류를 모두 「전체」로 두면 시간 역순 전체와 같습니다. 그 외에는 같은 대분류 안에서 질문을 최대 5개의 중분류로 자동 묶습니다(세션·시간대 기준).
