## 슬라이드 1 — 타이틀 (~15초)

### 화면에 넣을 것

- **제목**: 주제 인식형 행정 AI 업무지원 플랫폼
- **부제**: 연속 대화 · 자동 안건 · RAG · 법령 · 문서·검토 체인
- **이름 / 팀 / 캡스톤 과목명**

### 발표 대본 (예시)

안녕하세요. 저희는 **지자체 내부 직원**이 매일 겪는 **안건 정리·근거 찾기·법령 확인**을 줄이기 위해, **대화만 이어가도 업무 단위가 자동으로 정리되는** 행정 AI 플랫폼을 구현했습니다.

### 시각 자료 제안

- 로고 또는 지자체 업무 아이콘 1개 + 제품 이름만 크게 (잡음 없이).

### 데모 시나리오 (이 슬라이드에서는 생략)

- 인사만. 데모는 슬라이드 5로 미룸.

## 슬라이드 2 — 문제 (Pain) (~25초)

### 화면에 넣을 것 (불릿)

- 채팅·메일·문서가 **한데 섞임** → 나중에 **안건별로 꺼내기 어려움
- 내부 지침·보고서는 **여러 파일**에 흩어짐 → 답할 때 **근거 붙이기**가 번거로움
- 법령은 **별도 사이트 검색** → 질문 하나에 **시간·실수 비용**

### 발표 대본 (예시)

행정 현장에서는 질문과 자료가 **한 스레드에 쌓이지만**, 실제 업무는 **안건 단위**로 관리됩니다. 그런데 기존 챗봇은 **세션을 사람이 직접 나눠야** 하고, 답도 **내부 문서·법령과 연결되지 않은 경우**가 많습니다. 저희는 이 **전환 비용**을 없애는 데 집중했습니다.

### 시각 자료 제안

- Before 다이어그램: 질문 → (수동) 폴더 정리 → (수동) 법령 검색.

### 데모 시나리오 (연계)

- 다음 슬라이드에서 “After”로 바로 대비시키기 위해, 여기서는 **구체 제품명 언급 최소화**.

---

## 슬라이드 3 — 솔루션 (한 장) (~25초)

### 화면에 넣을 것 (불릿)

- **하나의 대화 스트림**에 메시지 저장
- 메시지마다 **자동으로 안건(토픽)** 부착 · 신규 생성
- **내부 문서 RAG** + (설정 시) **국가법령정보 연동**
- 필요 시 **보고서 초안**과 **검토·시뮬레이션 에이전트 체인**

### 발표 대본 (예시)

저희 시스템은 직원이 **새 세션을 만들지 않아도**, 대화가 이어질수록 **시스템이 안건을 자동 분류**합니다. 그 안건을 기준으로 **내부 지식을 검색**하고, 설정하면 **현행 법령 본문**까지 가져와 답에 반영합니다. 같은 맥락에서 **보고서 초안**까지 이어 갈 수 있습니다.

### 시각 자료 제안

- After 한 줄 플로우: 대화 → 자동 토픽 → RAG/법령 → 답변·문서.

### 데모 시나리오 (연계)

- “지금 화면에서 **같은 안건**과 **새 안건**이 어떻게 갈리는지** 보여드리겠습니다.” (슬라이드 5로 연결)

---

## 슬라이드 4 — 핵심 기능 3축 (~45초)

### 화면에 넣을 것 (3칼럼 또는 3블록)

1. **자동 안건(토픽)**
  - 메시지 단위 라우팅 · 기존 안건 연결 또는 신규 생성 · 병합·분리·제목 수정
2. **하이브리드 RAG**
  - BM25 + 임베딩 유사도 · 다중 문서 필터 · 응답에 근거(청크) 요약
3. **법령 + 문서 체인**
  - 법령 포커스 Intent 시 조회 강화 · (OC 설정 시) 검색·본문 파이프라인  
  - 보고서 작성 → 상급자 검토 → 법령 체크 등 **에이전트 체인**

### 발표 대본 (예시)

첫째, **토픽 매니저**가 매 메시지를 **업무 단위**에 붙입니다. 둘째, 검색은 **키워드와 의미 검색을 함께** 써서 행정 문서처럼 **조문 번호·고유명사**가 섞인 질문에도 대응합니다. 셋째, 법령은 **설정된 경우 실제 Open API**로 가져오고, 문서는 **한 번의 요청으로 다단계 검토**까지 돌릴 수 있게 API를 열어 두었습니다.

### 시각 자료 제안

- 간단한 블록 다이어그램(토픽 / RAG / Legal+Agents). 숫자·화살표만.

### 데모 시나리오 (연계)

- 데모에서 **반드시 보여줄 3포인트**를 입으로 예고:  
(A) 연속 질문으로 **같은 안건 유지**  
(B) **완전 다른 주제** 한 번 넣어 **새 안건**  
(C) **문서 선택** 후 질문 또는 **법령 관련 질문** 한 번

## 슬라이드 5 — 라이브 데모 (~50초)

### 화면에 넣을 것

- **데모 체크리스트** (발표자만 보는 노트용으로 PPT 노트에 복사 가능):
  1. `/ui` 접속 → 로그인 또는 데모 헤더
  2. 새 대화 또는 기존 스트림 선택
  3. 질문 A → 질문 B (같은 주제)
  4. 질문 C (다른 주제) → 사이드바·토픽 변화 언급
  5. (선택) 문서 업로드/선택 후 질문 D
  6. (선택) 보고서 생성 버튼 또는 API만 언급

### 발표 대본 (타임드; 읽으며 클릭)

“지금 **로컬에 띄운 UI**로 보여드리겠습니다. 먼저 **같은 안건**으로 이어지는지 확인하기 위해, ○○에 관해 질문하고 **바로 이어서** 세부 질문을 하겠습니다. … 이제 주제를 바꿔 **△△**을 물어보면, 시스템이 **새 안건**으로 분리하는 것을 확인할 수 있습니다. 내부 자료는 **문서를 선택한 뒤** 질문하면, 답변에 **근거로 쓰인 청크**가 붙는 흐름을 보여드리겠습니다.”

### 시각 자료 제안

- 슬라이드 본문은 짧게: **“Live Demo”** + 체크리스트 아이콘. 실제 화면은 노트북 투사.

### 데모 시나리오 (상세 스크립트 — 예시 문장만 교체해 사용)

**사전 조건**: `cap/backend`에서 서버 기동 완료. `OPENAI_API_KEY` 권장. 법령 데모는 `LAW_GO_KR_OC` 설정 시만 “실제 조회” 강조.


| 단계  | 행동                | 예시 입력 (행정 맥락)                                         |
| --- | ----------------- | ----------------------------------------------------- |
| 1   | UI에서 스트림 생성 또는 선택 | (UI 조작만)                                              |
| 2   | 같은 안건 유지          | 사용자: “○○ 사업 추진 일정이 어떻게 되나요?” → 이어서 “예산 집행은 언제까지인가요?”  |
| 3   | 안건 전환             | 사용자: “민원 응대 시 개인정보 알림 의무를 알려줘.” (앞 주제와 명확히 다르게)       |
| 4   | RAG               | 사이드바에서 업로드한 **내부 PDF/문서** 선택 후: “방금 올린 지침에서 절차 요약해줘.” |
| 5   | 법령 (OC 있을 때)      | “개인정보 보호법에서 정보주체 권리 관련 조항을 요약해줘.” + 답변 하단 부록·링크 가리키기  |
| 6   | 문서 (시간 남으면)       | “이 안건으로 보고서 초안 만들어줘” 또는 UI의 보고서 기능이 있다면 클릭            |


**시간 부족 시**: 단계 2~3만으로도 “자동 토픽” 스토리는 완결됩니다.

## 슬라이드 6 — 기술 스택 · 차별점 (~15초)

### 화면에 넣을 것 (불릿)

- **FastAPI** REST API · OpenAPI `/docs`
- **SQLAlchemy** + SQLite/Postgres, **Supabase Auth** 연동 설계
- **다중 LLM** (OpenAI / Anthropic / Gemini) + 작업·토픽별 모델 우선순위
- **보안**: 사용자 API 키 **Fernet 암호화** · **감사 로그**
- **법령 로직 TS 이식** (`cap/law-qa`)으로 규칙·테스트 재사용

### 발표 대본 (예시)

백엔드는 **FastAPI**로 모듈화했고, 인증은 **Supabase JWT**를 전제하되 로컬에서는 **데모 헤더**로도 돌릴 수 있게 했습니다. 모델은 한 곳에 묶지 않고 **작업별로 바꿀 수 있게** 해 기관 정책에 맞출 수 있습니다. 민감 정보는 **키 암호화**와 **감사 로그**로 최소한의 거버넌스를 갖췄습니다.

### 시각 자료 제안

- 로고 행: FastAPI, Supabase, (선택) OpenAI 등.

### 데모 시나리오

- 없음. 한 문장만: “상세는 README와 `cap/설명.md`에 정리했습니다.”
